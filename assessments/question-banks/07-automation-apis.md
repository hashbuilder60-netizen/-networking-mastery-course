# Q61-Q70: Automation and APIs

61. Q: Idempotent change means?
A: Repeated runs converge to same state.

62. Q: Why use source-of-truth inventory?
A: Reduce drift and manual inconsistency.

63. Q: Pre-check purpose?
A: Validate safe starting state.

64. Q: Post-check purpose?
A: Prove intended outcome and detect regressions.

65. Q: HTTP 200 always success?
A: No, intent may still not be applied.

66. Q: Why pilot subset rollout?
A: Limit blast radius.

67. Q: Rollback plan should be?
A: Tested before production change.

68. Q: Git in network ops gives?
A: Traceability and review history.

69. Q: API auth failures often caused by?
A: Expired or missing tokens.

70. Q: Best automation evidence artifact?
A: Before/after diff plus validation report.
