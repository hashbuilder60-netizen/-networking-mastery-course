# Capstone Project: Campus Network

## Scenario
A growing company needs a secure and scalable 3-floor campus network supporting employees, voice devices, printers, servers, guest Wi-Fi, and remote access.

## Requirements
- 300 users across 6 departments.
- Separate VLANs per department and service zone.
- Inter-VLAN routing with redundancy.
- ACL-based segmentation and least privilege policy.
- DHCP, DNS, NTP, and internet NAT.
- Wireless staff and guest SSIDs.

## Deliverables
- Topology diagram (logical + physical)
- IP and VLAN plan
- Device configurations
- Validation checklist and test evidence
- Incident simulation and RCA

## Success Criteria
- All intended communications work.
- Unauthorized paths are blocked.
- Core services are resilient.
- Documentation is complete and reproducible.
