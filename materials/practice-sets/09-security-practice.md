# Practice Set: Module 09: Network Security and Hardening

## Scenario A: Build

- Recreate the baseline topology for this module from memory.
- Implement required configs with clear interface descriptions.
- Validate connectivity and expected policy behavior.

## Scenario B: Break and Recover

- Inject one Layer 1 or Layer 2 fault.
- Inject one Layer 3 or service fault.
- Identify symptoms, isolate root cause, and restore operation.

## Scenario C: Evidence and Reporting

- Capture five verification outputs that prove healthy state.
- Summarize incident timeline and blast radius.
- Propose one preventive control to avoid recurrence.

## Time Targets

- Build and verify: 35 minutes.
- Troubleshoot and recover: 20 minutes.
- Report and recommendations: 15 minutes.

## Submission Checklist

- Topology diagram attached.
- Config snippets included.
- Verification evidence included.
- Root-cause explanation included.
- Preventive action item included.
