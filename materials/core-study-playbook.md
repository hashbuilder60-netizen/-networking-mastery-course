# Core Study Playbook

## Learning Workflow

1. Learn the model: understand protocol behavior and packet flow.
2. Build the config: apply concepts in lab topologies.
3. Verify deeply: confirm using commands and packet captures.
4. Break intentionally: inject failures to build troubleshooting instincts.
5. Explain clearly: document root cause and corrective action.

## Weekly Cadence

- Day 1: Theory + notes.
- Day 2: Guided lab.
- Day 3: Troubleshooting lab.
- Day 4: Practice set + review questions.
- Day 5: Timed mini practical.
- Day 6: Recap and memory recall.
- Day 7: Rest or light review.

## Documentation Discipline

For every lab, record:

- Topology and addressing map.
- Configuration snippets by device.
- Verification evidence.
- Failure symptoms.
- Root cause.
- Final fix and prevention control.

## Mastery Standard

You are ready to move on when you can:

- Explain "why" each protocol choice was made.
- Rebuild the lab from memory.
- Diagnose two injected failures under time pressure.
- Teach the concept to another learner in simple terms.
