# Contributing Guide

## Contribution Areas
- Improve explanations and diagrams
- Add lab scenarios and answer keys
- Add automation scripts and validation tooling
- Fix typos, factual errors, and broken links

## Standards
- Keep content technically accurate and testable.
- Use clear headings and short sections.
- Include prerequisites, steps, and expected results for every lab.
- For protocol behavior, explain both normal and failure cases.

## Pull Request Checklist
- [ ] Content is technically validated.
- [ ] Commands are copy/paste safe.
- [ ] Diagrams updated if topology changed.
- [ ] Quiz answers included in protected answer file.
- [ ] Module README checklist updated.
